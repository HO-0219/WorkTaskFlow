# PC·모바일 저해상도 와이어프레임

상태: Phase 0 목표 구조 참고. 로컬 알파는 상단 링크와 화면별 반응형 카드 셸로 단순화

## PC 공통 셸

```text
┌──────────────┬──────────────────────────────────────┬─────────────┐
│ Logo         │ 그룹 선택기          검색   알림  프로필 │ 상세 패널(선택)│
├──────────────┼──────────────────────────────────────┼─────────────┤
│ 홈           │ 페이지 제목                주요 행동 │ 요약/이력       │
│ 내 업무       │ 필터·탭·기간                         │ 댓글/체크리스트  │
│ 그룹          │                                      │             │
│ 캘린더        │ 주요 콘텐츠                          │             │
│              │                                      │             │
│ 설정          │ 로딩/빈 화면/오류도 이 영역에서 교체   │             │
└──────────────┴──────────────────────────────────────┴─────────────┘
```

상세 패널은 업무 빠른 확인에 사용하고, 편집·댓글·상태 변경처럼 중요한 작업은 `/tasks/:taskId` 전체 페이지에서도 동일하게 제공한다.

## 모바일 공통 셸

```text
┌──────────────────────────┐
│ 페이지 제목      알림  프로필 │
├──────────────────────────┤
│ 그룹 선택 / 필터           │
│                          │
│ 한 번에 하나의 주요 콘텐츠  │
│ 카드·리스트 단일 열         │
│                          │
├──────────────────────────┤
│ 홈  내 업무  그룹  캘린더  더보기│
└──────────────────────────┘
```

업무 등록과 상태 변경은 화면 하단 고정 행동 영역을 사용할 수 있다. 키보드가 열릴 때 입력란과 확정 버튼을 가리지 않아야 한다.

## 개인 대시보드

```text
[오늘의 요약: 오늘 마감 | 지연 | 진행 | 알림]
[내 우선 업무 목록                         ]
[그룹별 내 업무]         [다가오는 일정       ]
[최근 알림                                  ]
```

모바일에서는 요약 카드를 가로 스크롤하지 않고 2열 또는 단일 열로 재배치한다. 위험·지연 항목을 일반 통계보다 먼저 보여준다.

## 그룹 업무 목록

```text
[그룹명 / 업무] [목록] [상태] [담당자] [마감]     [+ 업무]
┌─────────────────────────────────────────────────┐
│ 상태 │ 업무명             │ 담당자 │ 마감 │ 우선순위 │
│ 요청 │ API 계약 검토       │ 미지정 │ D-2 │ 높음     │
│ 진행 │ 로그인 QA           │ 홍길동 │ 오늘 │ 보통     │
└─────────────────────────────────────────────────┘
[페이지네이션 또는 더 보기]
```

모바일 목록은 `업무명 → 상태/우선순위 → 담당자/마감` 순서의 카드로 바꾼다. 필터는 바텀시트로 열고 적용된 필터 수를 표시한다.

## 업무 상세

```text
[상태 배지] 업무 제목                         [편집]
설명
[담당자] [마감] [우선순위] [요청자]
[허용된 다음 상태 행동: 수락/반려 또는 시작/보류/완료]

[체크리스트 3/5]
[댓글·멘션]
[상태 변경 이력]
```

위험 행동은 확인 대화상자를 사용한다. 반려·보류·취소는 사유 입력 대화상자를 사용하며 빈 값으로 제출할 수 없다.

## 그룹과 멤버 관리

```text
[그룹 목록]                  [+ 그룹 만들기]
[그룹 카드: 역할 / 멤버 수 / 내 미완료 업무]

그룹 설정 > 멤버
[검색] [초대]
이름 | 그룹 역할 | 가입 상태 | 역할 변경/내보내기
```

`역할 변경`, `내보내기`, `설정`은 LEADER에게만 표시한다. 마지막 LEADER의 역할 변경·탈퇴 시도에는 위임 안내를 보여준다.

## 캘린더

```text
[오늘] [이전] 2026년 7월 [다음] [월/주/목록] [+ 일정]
[그룹 색상·업무 마감·일정 필터]
┌──────────────── 월간/주간 그리드 ────────────────┐
│ 일정 제목 + 구분 아이콘 + 그룹명                 │
└─────────────────────────────────────────────────┘
```

모바일 기본 보기는 일정 목록이다. 날짜를 선택하면 해당 날짜의 일정과 마감 업무가 아래에 표시된다.

## 알림 센터

```text
[알림]                              [모두 읽음]
[전체] [업무] [댓글] [그룹]
● 업무가 할당되었습니다.      그룹명 · 3분 전
  업무 제목 / 보낸 사람 또는 이벤트 요약
```

읽지 않음은 색상뿐 아니라 점과 텍스트 굵기로 구분한다. 선택하면 즉시 대상 화면으로 이동하되 이동 실패 안내도 제공한다.

## 공통 상태 와이어프레임

```text
로딩:  제목/카드 형태를 유지한 skeleton + 접근 가능한 상태 문구
빈 화면: 이유 + 다음 행동 1개 (예: “첫 업무 만들기”)
오류: 간단한 원인 + 다시 시도 + 필요 시 문의용 오류 코드
403: 접근 권한 없음 + 안전한 이전 위치로 이동
404: 대상을 찾을 수 없음 + 목록으로 이동
오프라인: 읽기 전용 표시 + 연결 후 재시도
```
