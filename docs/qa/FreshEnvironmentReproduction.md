# 새 개발 환경 설치·실행 재현 결과

검증일: 2026-07-22
범위: 로컬 MySQL 기반 로컬 알파. AWS·S3·Docker·CI/CD 제외

## 결론

Git 이력, 실제 `.env`, 백엔드·프런트엔드 빌드 산출물과 설치 의존성을 제외한 별도 작업 디렉터리에서 README 절차를 재현했다. 패키지 설치, 전체 테스트, 프로덕션 빌드, 시연 데이터 생성, 백엔드·프런트엔드 연결 및 빈 MySQL 스키마 초기화가 모두 성공했다.

## 검증 환경

- Java: 21.0.11
- Maven: Wrapper 3.9.11
- Node.js: 22.14.0
- npm: 10.9.2
- MySQL Server: 8.3
- 임시 작업 디렉터리에는 `.git`, `.env`, `backend/target`, `frontend/node_modules`, `frontend/dist`, TypeScript 빌드 캐시를 복사하지 않았다.

MySQL 8.3이 현재 프로젝트의 Flyway가 공식 확인한 최신 MySQL 8.1보다 새 버전이라는 업그레이드 권고 로그가 출력됐다. V1~V12 적용과 Hibernate 검증은 모두 성공했으며 로컬 알파를 막는 오류는 아니다. 의존성 갱신 시 Flyway 호환 범위를 다시 확인한다.

## 재현 중 발견하고 해결한 문제

1. 실제 `.env`가 Git에서 제외되지만 복사할 안전한 예제 파일이 없었다.
   - 루트에 `.env.example`을 추가하고 DB, JWT, 메일, OAuth 설정 키를 안전한 예제값으로 제공했다.
   - README에 복사, 필수값 변경, DB 생성과 실행 순서를 기록했다.
2. 개인 대시보드 테스트가 오늘 18:00을 마감으로 사용해 18시 이후 실행 시 지연 업무로 판정됐다.
   - 오늘 마감 시각을 23:59:59로 고정해 실행 시각과 무관하게 같은 계약을 검증하도록 수정했다.
