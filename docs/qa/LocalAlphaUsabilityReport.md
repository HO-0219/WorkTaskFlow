# 로컬 알파 PC·모바일 사용성 QA 결과

실행일: 2026-07-22
범위: AWS·S3·Docker·CI/CD 제외, 로컬 MySQL과 프로덕션 프런트 미리보기

## 환경과 방법

- Chrome 헤드리스 디바이스 에뮬레이션으로 모바일 `360×800`, PC `1280×900`을 측정했다.
- 로컬 MySQL 개발 DB와 Spring Boot `8081`, Vite preview `5174`를 실제로 연결했다.
- UI QA 전용 계정으로 로그인해 보호 화면을 확인한 뒤 계정을 탈퇴 처리하고 브라우저 토큰을 제거했다.
- 각 화면에서 `documentElement.scrollWidth`와 viewport 폭을 비교하고 보이는 링크·버튼·입력의 높이를 측정했다.

## 확인 화면

- 홈, 로그인, 회원가입
- 프로필, 계정 설정
- 그룹 목록·상세
- 업무 목록·등록
- 알림, 캘린더
- 개인 그룹 대시보드

## 결과

- 10개 핵심 화면 모두 360px와 1280px에서 문서 폭이 viewport와 일치해 가로 스크롤이 발생하지 않았다.
- 인증 입력, 그룹·업무 카드와 폼이 360px 화면 안에 배치됐다.
- 모든 보이는 텍스트 링크와 일반 버튼의 최소 터치 높이를 44px로 통일했다.
- 캘린더의 이전 달·오늘·다음 달 버튼도 44px로 보정했다.
- 라우트 이동 시 `#main-content`로 포커스가 이동하고 본문 건너뛰기 링크를 직접 포커스할 수 있음을 확인했다.
- 로컬 preview의 `/api` 요청이 백엔드 로그인까지 전달되어 `200` 응답을 받는 것을 확인했다.
- 프런트 프로덕션 빌드는 61 modules transformed로 성공했다.

## 발견 및 수정

1. 텍스트 링크의 높이가 글자 높이인 19~24px에 머물렀다. 공통 링크를 44px 터치 영역으로 확장했다.
2. 캘린더 월 이동 버튼이 별도 규칙 때문에 36px였다. 최소 높이를 44px로 변경했다.
3. Vite preview의 API 프록시 설정을 명시해 개발 서버와 동일한 로컬 백엔드 연결 기준을 고정했다.

실기기 설치 UI, 브라우저별 PWA 설치 동작과 200% 확대의 육안 확인은 새 환경 재현 단계에서 다시 확인한다.
