# 로컬 알파 시연 데이터와 발표 순서

## 데이터 준비

로컬 MySQL과 루트 `.env`를 준비한 뒤 프로젝트 루트에서 실행한다.

```bash
cd backend
./scripts/seed-demo-data.sh
```

이 명령은 `localhost` 또는 `127.0.0.1`의 `teamProject` DB에서만 실행된다. 기존 일반 사용자 데이터는 유지하고 아래 `demo_` 계정과 해당 계정이 만든 데이터만 초기화한다. 발표 도중 데이터를 변경했을 때 같은 명령을 다시 실행하면 기준 상태로 돌아온다.

## 시연 계정

세 계정의 공통 비밀번호는 `demo1234!`이다.

| 계정 | 역할 | 주요 확인 내용 |
|---|---|---|
| `demo_leader` | 팀장 김팀장 | 승인·담당 지정, 그룹 설정, 캘린더, 팀 대시보드 |
| `demo_member` | 팀원 이팀원 | 업무 제안, 담당 업무, 체크리스트, 댓글·멘션 알림 |
| `demo_viewer` | 팀원 박팀원 | 일반 멤버 권한, 보류 업무, 상태 변경 알림 |

## 기준 데이터

- 개인 그룹 3개와 `로컬 알파 시연팀` 1개
- 팀 업무 6개: 승인 대기·할 일·진행 중·보류·완료·지연
- 김팀장 개인 업무 1개
- 진행 중 업무의 체크리스트 3개와 댓글 2개, 멘션 1개
- 읽음·읽지 않음 알림과 팀·개인 일정
- 상대 날짜를 사용하므로 실행 시점 기준의 오늘 마감·지연·향후 일정이 유지된다.

## 권장 발표 순서

1. `demo_leader`로 로그인해 홈 대시보드의 지연·진행·알림 수를 보여 준다.
2. 그룹 목록에서 `로컬 알파 시연팀`으로 이동해 멤버 3명과 팀장 권한을 확인한다.
3. 팀 업무 목록에서 승인 대기·진행 중·보류·완료·지연 상태를 비교한다.
4. `모바일 화면 최종 점검` 업무에서 상태 이력, 체크리스트 2/3 완료, 댓글과 멘션을 보여 준다.
5. 팀 캘린더에서 직접 일정과 업무 마감일이 함께 표시되는 것을 보여 준다.
6. 팀 대시보드에서 상태별 수, 위험 업무와 팀원별 집계를 확인한다.
7. `demo_member`로 다시 로그인해 팀장 전용 입력이 숨겨지고 담당 업무를 변경할 수 있음을 보여 준다.
8. 알림 센터에서 댓글 멘션 알림을 열어 업무 상세로 이동한다.

승인·상태 변경·체크리스트 완료 등으로 기준 데이터가 바뀌면 `./scripts/seed-demo-data.sh`를 다시 실행한다.

## 검증된 기준

- 스크립트 연속 2회 실행 결과가 동일했다.
- 세 계정 모두 로그인 `200`을 확인했다.
- 팀장 API 기준 그룹 2개, 팀 업무 6개, 팀 캘린더 합성 항목 8개, 알림 1개가 조회됐다.
- 팀 대시보드에서 업무 6개와 멤버 3명이 집계됐다.
- 진행 중 업무에서 체크리스트 3개(완료 2개)와 댓글 2개가 조회됐다.
