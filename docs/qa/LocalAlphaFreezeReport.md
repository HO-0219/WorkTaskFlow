## 미해결 개선 항목

### Medium

1. 만료된 access token을 모든 화면에서 자동 refresh하지 않는다.
   - 영향: 사용자는 `로그인이 필요합니다.` 오류를 보고 홈 이동 또는 재로그인이 필요하다.
   - 현재 안전성: 보호 API가 `401 AUTHENTICATION_REQUIRED`로 차단되며 데이터 손상은 없다.
   - 동결 결정: 전역 재시도·동시 refresh 제어는 변경 범위가 커 로컬 알파 이후 처리한다.
2. 극히 동시에 발생한 캘린더·댓글·체크리스트의 JPA 낙관적 락 예외는 공통 fallback에서 `TASK_VERSION_CONFLICT` 코드가 될 수 있다.
   - 영향: HTTP `409`와 새로고침 안내는 정확하지만 자원 이름이 업무로 표시될 수 있다.
   - 현재 안전성: 쓰기 충돌은 차단되고 데이터는 보존된다.
   - 동결 결정: 기능 차단이나 데이터 손상이 없어 자원별 동시성 강화 작업으로 이관한다.

### Low / 후속 기능

1. 프로필 이미지는 파일 업로드가 아니라 URL 입력만 지원한다.
2. 관리자용 계정 정지·복구 화면은 제공하지 않는다.
3. 일반 계정과 소셜 계정의 명시적 연결 화면은 제공하지 않는다.
